Problem statement

we are about to turn on auto-deploy to prod when merging on master, but we want more confidence that we are not merging broken code.

Proposed solution

Smoke tests on every page that remder the page with mocked data and ensures there are no

console.error

Why?

  • they are fast, so they will not be a hassle to wait for before merging
  • they are not flacky, so we will not be flooded by false positives like we are now
  • they only test our code (not the integration with backend/3rd party services), they serve a more specific purpose
  • they are easy to write

What do we need to turn this into a standard?

  • every endpoint exposed by `react-sdk-api` should also expose a mock of the returned data
  • we should have a unique "mock all" function that set cache for all queries and is called before running the tests
  • we should write the tests (could probably even be done automatically through route generation)
